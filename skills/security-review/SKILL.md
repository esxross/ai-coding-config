# Security Review スキル

## 概要

PR差分（git diff）に限定したセキュリティチェックを実行します。
**信頼度スコア 0.8 以上の指摘のみを報告**することで、偽陽性を最小化します。

ファイルの編集は一切行わず、セキュリティ問題の検出とレポートに専念します。

## 引数

| 引数 | 説明 | 省略時 |
|------|------|--------|
| `base` | 比較元ブランチ | `main` または `master` |
| `head` | レビュー対象ブランチ | 現在の作業ブランチ |

## 重要な制約

**STRICTLY READ-ONLY**: ファイルの作成・編集・削除を一切行わない。
使用を許可するコマンドは以下のみ：
- `git diff`, `git log`, `git fetch`, `git show`, `git grep`
- `cat`, `find`（読み取り専用）
- ファイル読み取りツール

---

## 実行手順

### Step 1: 差分の取得

```bash
# PRレビュー（ブランチ指定あり）
git fetch origin
git diff origin/{base}...HEAD

# ローカル作業中（ブランチ指定なし）
git diff main...HEAD
```

差分が空の場合は直近コミットを対象にする：

```bash
git diff HEAD~1 HEAD
```

差分取得後、変更ファイル一覧とその種別（backend / frontend / config / infra）を把握する。

### Step 2: セキュリティ分析

以下の OWASP Top 10 カテゴリ別に差分を精査する。

| カテゴリ | チェック内容 |
|---------|-------------|
| A01 アクセス制御の不備 | 認証チェックの欠落、権限検証のバイパス、直接オブジェクト参照 |
| A02 暗号化の失敗 | 平文での機密データ送受信、弱いハッシュ（MD5/SHA1）、HTTP通信 |
| A03 インジェクション | SQLi, XSS, コマンドインジェクション, テンプレートインジェクション, LDAPi |
| A04 安全でない設計 | レート制限なし、セキュリティ機能の設計欠陥 |
| A05 セキュリティ設定ミス | デフォルト認証情報、デバッグモード露出、エラー詳細の漏洩 |
| A06 脆弱なコンポーネント | 既知脆弱性のある依存関係の追加（package.json等の変更を確認） |
| A07 認証・セッション管理の不備 | セッション固定、安全でないトークン生成、不十分なパスワードポリシー |
| A08 ソフトウェアの整合性の失敗 | 検証なしの外部データ使用、安全でないデシリアライズ |
| A09 不十分なロギング | 認証失敗・重要操作のログ欠落、個人情報のログ出力 |
| A10 SSRF | 外部URLの無検証フェッチ、内部ネットワークへのリクエスト |

追加でチェックする項目：
- ハードコードされた APIキー・パスワード・トークン・シークレット
- 安全でない乱数生成（`Math.random()` 等をセキュリティ用途に使用）
- 過度に許可的な CORS 設定（`*` など）
- `eval()` / 動的コード実行
- パスの検証なしのファイル操作

### Step 3: 信頼度スコアの付与

各指摘について以下の基準でスコアを付ける：

| スコア | 基準 | 例 |
|--------|------|----|
| 1.0 | コードに脆弱性が明確に存在する | SQLクエリへの文字列連結 |
| 0.9 | ほぼ確実に問題 | ハードコードされたAPIキー |
| 0.8 | 問題の可能性が高い | 入力検証なしの外部データ使用 |
| 0.7以下 | 文脈依存・要確認 | （報告しない） |

**信頼度 0.8 未満の指摘は出力しない。**

文脈を読んで判断する：既存のバリデーション層・フレームワークの保護機構が適切に機能している場合は、
スコアを下げるか報告しない。

## サマリー

全体評価と最も優先すべき修正点のまとめ。
報告が0件の場合はその旨を明示し、チェックしたカテゴリ数と変更ファイル数を示す。

---

## 重大度の定義

| レベル | 基準 |
|--------|------|
| CRITICAL | 直接的な攻撃が可能・機密データ漏洩・認証バイパス |
| HIGH | 悪用の可能性が高い・認証/認可の欠陥・SQLi/XSS |
| MEDIUM | 条件付きで悪用可能・設定ミス・情報漏洩リスク |

## カバレッジの限界

このスキルはコードの静的パターンを検査します。以下は検出できません：
- ビジネスロジックの欠陥（ペネトレーションテストが必要）
- ランタイム挙動の脆弱性
- 依存関係の既知CVE（`npm audit` / `trivy` 等を併用）
