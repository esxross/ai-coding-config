# Suggest Automations スキル

## 概要

過去5営業日の活動を横断分析し、繰り返しパターンを検出して**自動化候補を提案**するスキルです。
週次（毎週金曜 17:00 等）で実行することで、自分では意識していない定常作業に気づけます。

**推奨実行頻度**: 週1回（金曜午後）
**推奨スケジューラー**: Desktop → 安定後に Cloud へ昇格

---

## 実行手順

### Step 1: 活動データの収集

過去5営業日の活動を以下のソースから収集する。
利用可能な MCP のみ実行し、未接続のものはスキップする。

```
収集対象（利用可能なものを使う）:
- Slack:           自分の投稿・反応・DM送信の一覧
- Notion:          作成・編集したページ・タスクの一覧
- Google Calendar: 参加した MTG・予定の一覧
- GitHub:          作成・レビューした PR・Issue の一覧
- ローカルファイル:  git log で作業ログを確認
```

### Step 2: 繰り返しパターンの検出

収集したデータから以下のパターンを探す。

**検出すべきパターン**:
- 毎日同じチャンネルへの手動投稿
- 定例 MTG 前後の準備・後処理作業
- 毎週同じ曜日・時間帯に発生する作業
- 同じフォーマットで繰り返し作成されるドキュメント
- 複数ツールをまたいだ定型ワークフロー（例: MTG → 議事録 → Slack 共有）

**スキップすべきもの**:
- 判断が必要な作業（コードレビューのコメント内容など）
- コンテキストに強く依存する意思決定
- 1回限りのプロジェクト作業

### Step 3: 自動化候補の評価

検出したパターンを以下の基準でスコアリングする。

| 評価軸 | 高評価の条件 |
| --- | --- |
| 繰り返し頻度 | 週3回以上 |
| 手順の明確さ | 毎回同じ操作・判断不要 |
| 自動化の難易度 | 接続済み MCP で完結する |
| 時間削減効果 | 1回あたり5分以上かかる |

### Step 4: 既存スケジューラーとの重複確認

現在稼働中のスケジューラーを確認し、既に自動化済みの作業を除外する。

```
確認場所:
- Desktop: Claude Desktop app のスケジューラー画面
- Cloud:   claude.ai/settings/scheduled-tasks
```

### Step 5: レポート出力

下記フォーマットでレポートを出力する。


## セットアップ（週次スケジューラーとして登録する場合）

### Desktop スケジューラーへの登録

Claude Desktop app でスケジューラーを新規作成：

```
名前:    suggest-automations
実行時刻: 毎週金曜 17:00
プロンプト:
  /suggest-automations を実行してください。
  結果を Slack DM（または ~/reports/automations/）に出力してください。
```

### Cloud 昇格条件

`rules/workflow/scheduler.md` のチェックリストを満たしたら Cloud に昇格する。
目安は Desktop で3週間連続して期待通りの出力が得られた時点。

---

## 活用のヒント

- 提案された候補はすべて自動化する必要はない。判断が必要なものは手動のまま残す
- 自動化の選定基準は `rules/workflow/scheduler.md` を参照
- 自動化したスキルは `skills/` に追加し、このリポジトリで管理する
