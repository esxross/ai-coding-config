# Git運用ルール

## コミットメッセージ（Conventional Commits）
- feat: 新機能
- fix: バグ修正
- docs: ドキュメントのみの変更
- refactor: リファクタリング
- test: テスト追加・修正
- chore: ビルド・設定変更

## PR作成
- タイトルはConventional Commits形式に従う
- セルフレビュー後にdraftを外す
- UI変更時はスクリーンショット必須
- レビュワーへの補足はPR本文に記載する

## ブランチ命名
- feature/[issue番号]-[簡潔な説明]
- fix/[issue番号]-[簡潔な説明]
