# Notion Sync Manager Skill

このスキルは、プロジェクトの知識、タスク、および仕様をNotion上で統合管理するためのガイドラインを提供します。コードベースの変更をNotionドキュメントに反映させたり、Notion上のフィードバックを開発に活かしたりすることを目的としています。

## 機能 (Capabilities)

1.  **コンテンツ管理**: ページの作成、更新、複製、およびNotion特有の「Enhanced Markdown」を用いたリッチなドキュメント作成。
2.  **データベース操作**: データベースの作成、スキーマ変更、および特定条件に基づくクエリ。
3.  **コラボレーション連携**: ページや特定のブロックに対するコメントの取得および作成。
4.  **リソース検索**: ワークスペース全体や接続された外部ソース（Slack, Drive等）からのセマンティック検索。

## 使用方法 (Usage Instructions)

### 1. ドキュメントの同期 (Sync to Notion)

コードベースの重要な変更（新機能の実装、APIの変更など）があった際、関連するNotionページを更新します。

- **新規ページ作成**: `create_pages` を使用し、リッチなMarkdown形式で仕様を記述します。
- **特定箇所の更新**: `replace_content_range` や `insert_content_after` を使い、既存ドキュメントの整合性を保ちながら情報を追記します。

### 2. データベースの活用 (Database Operations)

試験問題の管理やバグトラッキングなど、構造化されたデータを扱う際に利用します。

- **スキーマ管理**: `update_data_source` でプロパティの追加や変更を行います。
- **データ登録**: インポートするデータをNotionのデータベース形式に整形し、一括で登録します。

### 3. 情報の探索 (Gathering Context)

実装前に仕様を確認したり、過去の議論を参照したりします。

- **全文検索**: `search` (query_type: "internal") を使用して、プロジェクトに関連する既存情報をワークスペース全体から探し出します。
- **詳細取得**: ページURLやIDから `fetch` を行い、実装のヒントとなるコンテキスト（Enhanced Markdown）を抽出します。

## 運用ルール (Rules) 🔴

1.  **フェッチ優先**: ページやデータベースを更新する際は、必ず事前に `fetch` を行い、現在のスキーマやコンテンツ構造を確認すること。
2.  **Markdown仕様の遵守**: Notion独自のMarkdown拡張（コールアウト、トグル等）を適切に使用し、視認性の高いドキュメントを目指すこと。
    - 詳細は `notion://docs/enhanced-markdown-spec` を参照。
3.  **子ページの保護**: `replace_content` 等でページ全体を置換する際、誤って子ページやサブデータベースを削除しないよう注意すること。
4.  **一意なセグメント指定**: コンテンツを部分置換する際は、`selection_with_ellipsis` を使用して一意な箇所を指定すること。

## 指示出しパターン (Instruction Patterns)

- **新規作成**: 「[このディレクトリ/ファイル]の仕様を、Notionの『[親ページ名]』の下に新しいページとして書き出して」
- **仕様更新**: 「[ファイル名]の最新の変更内容を反映するように、Notionの該当ページを更新して」
- **検索依頼**: 「この機能の背景について、Notion上の過去のドキュメントから探して概要を教えて」
